# Contributing

Thank you for your interest in PlanNote. This is an early-stage project, so small improvements are especially helpful.

## Good First Contributions

- Improve documentation and setup instructions
- Add screenshots or demo assets
- Fix UI bugs
- Improve mobile layout
- Add tests or quality checks
- Suggest AI-assisted planning workflows
- Improve security documentation

## Local Development

```bash
npm install
npm run dev
```

Before opening a pull request, run:

```bash
npm run build
```

## Pull Request Guidelines

- Keep changes focused and easy to review
- Explain the user-facing problem being solved
- Include screenshots for UI changes when possible
- Do not commit secrets, API keys, or personal reminder keys
- Update documentation when behavior changes

## AI Feature Contributions

AI-related changes should be practical and transparent. Please describe:

- What user problem the feature solves
- What data is sent to the model
- How the response is used in the app
- What fallback exists when the API is unavailable
