# GitHub Star Growth Plan

This plan focuses on realistic, ethical ways to make PlanNote easier to understand, try, and share.

## Make The Repository Easier To Trust

- Add a short repository description: `Local-first personal planning app with tasks, notes, calendar views, reminders, and planned AI assistance.`
- Add topics: `react`, `typescript`, `vite`, `tailwindcss`, `pwa`, `productivity`, `planning`, `notes`, `calendar`, `openai`
- Add a license so GitHub displays the project as clearly open source
- Remove committed secrets and rotate any exposed key
- Add screenshots or a short demo GIF near the top of the README

## Make The Project Easier To Try

- Publish a Vercel demo and link it in the repository website field
- Add a one-click deploy link later if the setup stays simple
- Keep quick start instructions short and accurate
- Add a small sample workflow in the README

## Make The Project Easier To Share

- Write one short launch post in Chinese and one in English
- Share the project in relevant communities after the README and demo are ready
- Explain the AI roadmap clearly instead of only saying "AI planning"
- Add screenshots showing the task list, calendar, notes, and reminder settings

## Suggested Launch Message

I am building PlanNote, a local-first personal planning app for tasks, notes, calendar views, and reminders. The next step is adding practical AI features such as task breakdown, note summarization, daily planning, and reminder drafting. Feedback and stars are welcome.

https://github.com/Alpha-lay/plan-note

Day 1: Update README, license, security docs, and repository description.

Day 2: Add screenshots and a hosted demo link.

Day 3: Create issues for AI task breakdown, note summarization, reminder drafting, and security cleanup.

Day 4: Post a short project update on X, LinkedIn, V2EX, Reddit, or relevant developer groups.
